이소티논(Isotretinoin) 복용 관리 앱의 메인 화면(홈 탭) UI/UX 상세 계획을 제안해 드립니다.

사용자가 앱을 켜자마자 **"오늘 약을 먹어야 하는 날인가?"**를 0.1초 만에 파악하고, **"먹었다"**는 기록을 가장 빠르고 직관적으로 할 수 있도록 설계하는 것이 핵심입니다.

📱 앱 구조 개요
하단 탭바 (Bottom Navigation Bar):

[좌측] 캘린더 (기록 조회)

[중앙] 홈 (오늘의 복용 체크 & 상태) - Current Focus

[우측] 커뮤니티 (정보 공유)

🏠 메인 화면 (홈) 상세 UI/UX 계획
화면 구성을 위에서 아래로(Top-down) 내려가며 배치합니다.

1. 상단 헤더 (Header)
   구성: 날짜 표시 / 설정(톱니바퀴) 아이콘

UX:

오늘 날짜를 크게 보여주어 사용자가 날짜를 착각하지 않도록 합니다 (예: 12월 11일 목요일).

우측 상단 설정 아이콘을 통해 알림 설정이나 프로필 수정으로 이동합니다.

2. 복용 상태 알림 카드 (Status Indicator)
   기능: 사용자가 설정한 주기에 따라 오늘이 '복용일'인지 '휴약일'인지 알려줍니다.

UI 디자인:

복용일인 경우: 밝고 눈에 띄는 색상(예: 오렌지 또는 블루) 배경의 카드.

텍스트: "오늘은 약 먹는 날이에요! 💊"

휴약일인 경우: 차분한 회색조 또는 파스텔톤 배경.

텍스트: "오늘은 쉬는 날입니다. 피부도 쉬어가요 🌿"

UX 로직:

사용자가 설정한 주기(매일/격일 등) 알고리즘에 따라 자동으로 텍스트와 배경색이 변합니다.

3. 메인 액션 버튼 (Main Check-in Action)
   기능: 약 복용 여부 체크 (Toggle)

UI 디자인: 화면 중앙에 위치한 가장 큰 인터랙티브 요소.

복용 전: 채도가 낮은 약 알약 아이콘이나 빈 동그라미. (텍스트: "복용 체크하기")

복용 후: 컬러풀한 알약 아이콘이나 꽉 찬 동그라미 + 체크 표시 (✔). (텍스트: "복용 완료!")

UX/인터랙션:

터치 시: 단순한 클릭보다는 살짝 꾹 누르거나, 탭 했을 때 기분 좋은 **햅틱 피드백(진동)**과 함께 폭죽(Confetti) 애니메이션이나 버튼이 '통통' 튀는 효과를 주어 복용에 대한 보상감을 줍니다.

실수로 눌렀을 경우를 대비해, 다시 누르면 체크가 해제됩니다.

4. 복용 주기 설정 영역 (Frequency Setting Display)
   기능: 현재 설정된 주기를 보여주고, 터치 시 변경할 수 있는 UI를 호출합니다.

UI 디자인: 메인 액션 버튼 하단에 작고 깔끔한 캡슐 형태의 버튼.

텍스트 예시: 현재: 격일 복용 (2일에 1알) ✏️

UX (Bottom Sheet 호출):

이 버튼을 누르면 화면이 전환되지 않고, **바텀 시트(Bottom Sheet)**가 하단에서 부드럽게 올라옵니다. (사용자가 맥락을 잃지 않게 하기 위함)

⚙️ 기능 상세: 복용 주기 선택 UI (바텀 시트 내부)
사용자가 위의 '복용 주기 설정 영역'을 눌렀을 때 나오는 화면입니다.

타이틀: "복용 주기를 선택해주세요"

선택지 (Radio Button 또는 Card List 형태):

매일 복용 (Daily)

격일 복용 (2일에 1알)

3일에 1알

일주일에 1알 (주 1회)

직접 입력 (커스텀) - (나중에 확장을 위해 고려)

기준일 설정 (DatePicker):

격일이나 3일 1회 복용의 경우, **"언제를 기준으로 계산할까요?"**라는 질문과 함께 '오늘부터' 또는 '어제부터' 등을 선택하거나 마지막으로 먹은 날짜를 선택하게 합니다. (이 날짜를 기준으로 오늘이 먹는 날인지 계산)

저장 버튼: "설정 완료"
