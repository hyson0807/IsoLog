복용을 체크하면 첨부한 이미지와 같이 크게 체크 버튼과 복용완료 멘트가 뜨고 있는데,

복용 체크를 하면 해당 섹션을 없앨려고해. 굳이 필요없는거 같아. 해제는 캘린더 페이지에서 할수 있으니까.

그러면 빈공간이 생기는데 거기에 어떤것들을 넣으면 좋을까?

아래 방법으로 결정완료!

트러블 상태: (잠잠해요 ✨) - (몇 개 났어요 🥲) - (심해졌어요 🚨)

건조함 정도: (촉촉해요 😌) - (보통이에요 🙂) - (건조해요 🌵)

(선택) 짧은 한 줄 메모 입력창

을 기록할수 있는 섹션이 나오게끔 하자.

그리고 캘린더페이지의 시트에서 확인이 가능하게끔 하자.



UI/UX 계획 구상해줘


좋습니다. \*\*"복용 체크 -\> 피부 상태 기록"\*\*으로 이어지는 흐름은 사용자가 앱에 더 애착을 갖게 만드는 훌륭한 루틴입니다.

메인 화면의 **[복용 완료]** 상태 UI와, 이것이 \*\*[캘린더 탭]\*\*에 어떻게 연동될지 상세한 UI/UX 계획을 그려드릴게요.

-----

### 1\. 🏠 메인 화면 (복용 체크 후 변경되는 UI)

사용자가 복용 버튼을 누르면, 거대한 체크 버튼이 사라지고 그 자리에 부드러운 애니메이션과 함께 \*\*[오늘의 피부 기록 카드]\*\*가 나타납니다.

#### A. 레이아웃 구조 (위에서 아래로)

1.  **상단 메시지 (유지):** "오늘 복용을 완료했어요\! 🎉" (배경색: 파스텔 그린/블루)
2.  **기록 카드 (메인 콘텐츠):**
    * **타이틀:** "오늘 피부 컨디션은 어떤가요?" (작고 부드러운 폰트)
    * **섹션 1: 트러블 상태 (3단계 선택지)**
    * **섹션 2: 건조함 정도 (3단계 선택지)**
    * **섹션 3: 한 줄 메모 (입력창)**
3.  **하단:** 주기 설정 바 (유지)

#### B. 컴포넌트 디자인 상세

가장 중요한 건 \*\*"기록이 숙제처럼 느껴지지 않게 하는 것"\*\*입니다. 클릭 한 번으로 끝나는 **'칩(Chip) 버튼'** 스타일을 추천합니다.

* **1 & 2. 상태 선택 버튼 (Choice Chips)**

    * 가로로 3개의 버튼을 배치합니다.
    * **선택 전:** 회색 테두리, 연한 회색 텍스트.
    * **선택 후:** 해당 상태를 상징하는 **파스텔톤 배경색**이 채워지며 테두리가 진해짐.
    * **디자인 예시:**
        * **트러블:** `[✨ 잠잠해요]` `[🥲 몇 개 났어요]` `[🚨 심해졌어요]`
        * **건조함:** `[😌 촉촉해요]` `[🙂 보통이에요]` `[🌵 건조해요]`

* **3. 한 줄 메모**

    * **Placeholder (힌트):** "특이사항이 있다면 메모해 주세요 (선택)"
    * **UI:** 옅은 회색 배경의 둥근 입력창.
    * **UX:** 키보드가 올라오면 화면이 살짝 위로 밀려 입력에 방해되지 않도록 처리(Keyboard avoidance).

#### C. 저장 방식 (UX Flow)

* **자동 저장:** 별도의 [저장하기] 버튼을 두지 않습니다. 사용자가 버튼을 누르거나 메모를 입력하고 키보드를 내리는 순간 자동으로 내부 DB에 저장됩니다. (가장 매끄러운 경험)

-----

### 2\. 📅 캘린더 탭 (기록 확인 및 수정)

캘린더에서 날짜를 눌렀을 때 올라오는 \*\*바텀 시트(Bottom Sheet)\*\*에 이 정보가 통합되어야 합니다.

#### 바텀 시트 UI 구조

기존에 계획했던 '복용 수정' 기능 아래에 '피부 기록' 섹션을 추가합니다.

1.  **헤더:** `12월 11일 (목)`
2.  **복용 여부 섹션:**
    * `[💊 복용 완료]` (스위치 또는 버튼 형태, 수정 가능)
    * (술 약속이 있는 날이라면 여기에 🍺 아이콘 표시)
3.  **피부 기록 섹션 (View Mode):**
    * 회색 박스 또는 카드 형태로 구분감을 줍니다.
    * **데이터가 있을 때:** 사용자가 선택한 이모지와 텍스트를 보여줍니다.
      > **피부 기록** 📝
      > • 트러블: 몇 개 났어요 🥲
      > • 건조함: 건조해요 🌵
      > • "어제 술 마셔서 그런지 좀 푸석함..."
    * **데이터가 없을 때:** "기록된 피부 상태가 없어요." 문구와 `[기록하기]` 텍스트 버튼 노출.

#### 수정 기능 (Edit Mode)

* 캘린더 바텀 시트 우측 상단이나 피부 기록 섹션 옆에 작은 **`[수정]`** 버튼(또는 연필 아이콘)을 둡니다.
* 누르면 바텀 시트 내부의 텍스트가 **메인 화면과 동일한 선택 버튼(Chips)과 입력창**으로 변하여 과거의 기록도 수정할 수 있게 합니다.

-----

### 3\. 🎨 시각적 피드백 (캘린더 달력 화면)

캘린더의 달력(월간 뷰)에서도 피부 상태를 아주 작게 힌트로 보여주면 좋을까요?

* **제안:** 너무 복잡해질 수 있으니, 우선은 \*\*'메모가 있는 날'\*\*만 날짜 칸 구석에 아주 작은 **회색 점(Dot)** 하나를 찍어주는 것을 추천합니다.
* 사용자는 "어? 점이 있네? 뭔가 적어놨었지" 하고 눌러보게 됩니다.
