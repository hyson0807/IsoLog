# Git Commit Helper

현재 작업 내용을 분석하고 커밋해주세요.

## 수행할 작업

1. `git status`로 변경된 파일 확인
2. `git diff --staged`로 스테이징된 변경사항 확인 (없으면 `git diff`로 전체 변경사항 확인)
3. 변경 내용을 분석하여 다음 형식으로 커밋 메시지 작성:
   - 타입: feat, fix, docs, style, refactor, test, chore 중 선택
   - 형식: `타입: 간결한 설명 (한국어)`
   - 예시: `feat: 로그인 API 연동 추가`

4. 스테이징된 파일이 없으면 `git add -A`로 모든 변경사항 스테이징
5. 작성한 커밋 메시지로 `git commit` 실행
6. 커밋 결과 요약 출력

## 커밋 메시지 규칙

- 제목은 50자 이내
- 명령형으로 작성 (예: "추가", "수정", "삭제")
- 변경사항이 많으면 본문에 상세 내용 추가

## 사용자 확인

커밋 전에 다음을 보여주고 확인 요청:
- 변경된 파일 목록
- 제안하는 커밋 메시지
- 진행 여부 (y/n)
